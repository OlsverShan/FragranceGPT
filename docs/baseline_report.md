## 3. 基准方法详情

### 3.1 频率基线
- **方法**：香调 → 香韵共现计数（每个香调取前5个，按层次划分）
- **模型**：无（仅统计方法）
- **文件**：`baseline.py`
- **成本**：免费，在 CPU 上数秒内完成

### 3.2 LLM 零样本基线
- **模型**：DeepSeek-Chat (deepseek-chat)
- **提供商**：DeepSeek API
- **方法**：单轮提示词："给定这些香调，预测前调/中调/基调香韵，以 JSON 格式输出"
- **文件**：`llm_baseline.py`
- **评估样本数**：50（随机种子 42）
- **成本**：极低（约50次 API 调用 × 500 tokens）

---

## 4. 主要发现

### 4.1 LLM 在 F1 指标上优于频率基线（精确匹配 +12.7%，模糊匹配 +20.2%）
LLM 整体精确匹配 F1 = 0.337 vs 频率法 = 0.299。模糊匹配：0.393 vs 0.327。提升幅度虽不大但真实存在，且在所有三个层次中表现一致。

### 4.2 LLM 以召回率换取精确率（精确匹配）
| | 频率法 | LLM 零样本 | 方向 |
|---|--------|-------------|------|
| 精确率 | 0.212 | 0.293 | +38% |
| 召回率 | 0.562 | 0.438 | -22% |

LLM 更保守——它预测的香韵更少，但正确率更高：
- 频率基线："广撒网"——猜测大量香韵，捕获 56% 但仅有 21% 正确
- LLM："精挑细选"——谨慎选择香韵，29% 正确，捕获 44%

在模糊匹配下，差距缩小——频率法召回率升至 0.618，而 LLM 达到 0.514。

### 4.3 RAG 解决了精确率与召回率的权衡
相比零样本，RAG 同时提升了精确率（+19%）和召回率（+21%）（精确匹配）。检索到的参考香水为 LLM 提供了更广泛的每个香调的有效香韵词汇，使其预测既更准确又更全面。

### 4.4 基调始终比前调/中调更容易
基调（麝香、琥珀、檀香木、香草、广藿香）高度定型。所有方法的基调 F1 约为前调/中调的 1.5-2 倍。模糊匹配进一步拉大差距，因为基调拥有最多的修饰变体。

### 4.5 少样本是冗余的，多智能体是有害的
- 仅少样本相比零样本提升 +6.8%，但与 RAG 结合时无增益（0.406 vs 0.405）
- 多智能体编排使 RAG 下降 -14.8%，原因是 5 次顺序 LLM 调用中的错误传播

## 5. 基线 3：RAG（检索增强生成）

### 方法
- **向量存储**：ChromaDB，包含 24,063 个香水嵌入（all-MiniLM-L6-v2，余弦相似度）
- **检索**：对于每个查询香调集，检索前5个最相似的香水
- **注入**：将检索到的香水的真实前调/中调/基调香韵作为参考添加到 LLM 提示词中

### 结果（精确匹配）
```
层次    精确率   召回率   F1      相对零样本的变化
────────────────────────────────────────────────────
前调      0.248     0.419   0.300      +26.1%
中调      0.240     0.410   0.287      +27.0%
基调      0.392     0.534   0.431      +12.8%
总体      0.349     0.528   0.406      +20.5%
```

### 结果（模糊匹配）
```
层次    精确率   召回率   F1      相对零样本的变化
────────────────────────────────────────────────────
前调      0.256     0.436   0.311      +25.4%
中调      0.264     0.456   0.317      +26.3%
基调      0.432     0.584   0.475      -2.9%
总体      0.381     0.576   0.443      +12.7%
```

### 关键发现
RAG 同时提升了**精确率和召回率**（精确匹配：精确率 +19%、召回率 +21%；模糊匹配：+12%）——这与以召回率为代价换取精确率的零样本基线不同。检索到的参考香水为 LLM 提供了更广泛的每个香调的合理香韵词汇，使其预测既更准确又更全面。在模糊匹配下，RAG 达到 F1=0.443（精确率=0.381，召回率=0.576）。

## 7. 模糊香韵匹配（评估指标修正）

### 原因

旧版评估使用精确字符串匹配：
- "cedarwood" ≠ "cedar" → 被错误扣分
- "italian bergamot" ≠ "bergamot" → 被错误扣分
- "sandal" ≠ "sandalwood" → 被错误扣分

这些是相同的成分。模型因给出实质上正确、只是使用了不同有效名称的答案而被扣分。

### 我们做了什么

在 `utils.py` 中构建了一个三层模糊匹配回退系统：

**第1层 — 硬编码同义词映射（约120条）：**
手动整理了木材、麝香、柑橘、花香、香料和果香中最常见的变体。例如："cedarwood" → "cedar"、"vetivert" → "vetiver"、"white musk" → "musk"、"olibanum" → "frankincense"、"cassis" → "black currant"。

**第2层 — LLM 构建的同义词表（1,650条）：**
使用 DeepSeek 将所有 1,655 个独特香韵名称聚类为 1,421 个规范组。LLM 识别出 151 个具有 2+ 变体的组（例如 "acácia" → "acacia"、"agarwood (oud)" → "agarwood"）。保存至 `results/note_synonyms.json`。

**第3层 — 基于规则的规范化：**
- 去除地理前缀："italian bergamot" → "bergamot"、"sicilian lemon" → "lemon"
- 去除颜色修饰词（受保护的除外）："white musk" → "musk"，但 "black currant" 保持原样
- 去除后缀："rose absolute" → "rose"、"vanilla extract" → "vanilla"

**回退 — Levenshtein 编辑距离（阈值=2）：**
捕获规则遗漏的剩余接近拼写错误（例如 "graperfruit" → "grapefruit"）。

**关键设计决策：** 硬编码映射在规则规范化之前首先检查。受保护的短语列表防止 "black currant" 被错误地去除为 "currant"。

### 结果（所有方法）

| 方法 | 精确 F1 | 模糊 F1 | 增量 |
|------|----------|----------|------|
| 频率基线 | 0.299 | 0.327 | +9.4% |
| LLM 零样本 | 0.337 | 0.393 | +16.6% |
| LLM + RAG | 0.406 | 0.443 | +9.1% |
| RAG + 少样本 | 0.405 | 0.442 | +9.1% |

### 关键发现

1. **LLM 零样本从模糊匹配中获益最多**（+16.6%）——未辅助的 LLM 使用更具创造性/多样化的香韵名称，模糊匹配能正确规范化这些名称
2. **RAG 获益较少**（+9.1%），因为参考香水标准化了 LLM 的输出——它学会使用更规范的名称
3. **RAG 在绝对 F1 上仍然胜出**（0.443 vs 0.393）
4. **基调在所有方法中受益最大**——木材、麝香、树脂拥有最多的修饰变体
5. **少样本冗余性在两种指标下均得到确认**（精确 0.406 vs 0.405，模糊 0.443 vs 0.442）
6. **模型并未改变**——提升完全来自修正评估指标
7. **与 LLM 作为评判的教训相同**：在主观领域中，token 重叠不等于语义正确性

## 8. 微调 Qwen2.5-7B (QLoRA)

### 训练设置

| 参数 | 值 |
|------|-----|
| 基础模型 | unsloth/Qwen2.5-7B-Instruct-bnb-4bit |
| 微调方法 | QLoRA (r=16, alpha=16) |
| 量化 | 4-bit NF4 |
| 训练样本 | 22,859 |
| 评估样本 | 1,204 |
| 训练轮数 | 3 |
| 有效批次大小 | 16 (4 × 4) |
| 学习率 | 2e-4 |
| 可训练参数 | 40M / 7.6B (0.53%) |
| GPU | RTX 5080 16GB (vast.ai) |
| 训练时间 | ~1.5 小时 |
| 模型大小 | 154MB（适配器）+ 5.2GB（共享基础模型） |

### 性能表现

| 层次 | 精确匹配 P/R/F1 | 模糊匹配 P/R/F1 |
|------|----------------|----------------|
| 前调 | 0.293 / 0.248 / 0.257 | 0.293 / 0.248 / 0.257 |
| 中调 | 0.282 / 0.297 / 0.277 | 0.332 / 0.353 / 0.326 |
| 基调 | 0.427 / 0.360 / 0.366 | 0.483 / 0.407 / 0.415 |
| **总体** | **0.372** | **0.418** |

### 分析

**优势：**
- 优于 LLM 零样本（模糊匹配 +6.4%），与多智能体持平
- 完全免费且本地运行——无 API 成本、无延迟、无速率限制
- 单次前向传播（相比多智能体的 5 次 API 调用）
- 基调层从训练数据模式中受益最多

**劣势：**
- 无法匹敌 RAG（模糊匹配 -5.6%）——缺乏从香水数据库的实时检索能力
- 每层仅预测约 3 个香韵（按真实分布训练）
- 前调是最薄弱的层次（柑橘/清新类香韵高度多变）
- 仅限香调→香韵任务（无法进行通用对话）

**适用场景：**
- 演示、快速迭代、离线使用
- 作为免费的"足够好"预测器（F1=0.418）
- 与 RAG 结合以获得两全其美的效果
