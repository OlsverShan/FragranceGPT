项目：FragranceGPT — 基于 LLM 的智能评香与香氛推荐系统

为什么这个方向好

1. DSM-Firmenich 经历直接成为护城河——面试官追问领域知识时，你真有货
2. 感官评价天然适合 LLM——评香是模糊的、主观的、依赖自然语言描述的，这正是 LLM 的长处
3. 技术栈覆盖大厂面试所有关键词——RAG、Agent、结构化抽取、Evaluation、Fine-tuning
4. 有明确的量化评估指标——LLM 评香 vs 人类评香师的一致性、消费者偏好预测准确率
5. 全国独一无二——面试官看 100 份"做了个 RAG 问答系统"后，看到你的项目会停下来

各模块技术细节

模块 1：多源数据引擎

| 数据源 | 获取方式 | 处理方式 |
|--------|---------|---------|
| 香精配方（公开） | Good Scents Company 公开数据库、Fragrantica | 结构化解析，构建成分-香调映射 |
| 评香师笔记 | 你自己在 DSM 接触的真实评价范式 + 公开香水测评文本 | LLM 结构化抽取（三调、持久度、扩散性等维度） |
| GC-MS 报告 | 公开文献 | PDF → Markdown → embedding 向量 |
| 消费者评论 | Fragrantica/Basenotes 爬取 | 情感分析 + 属性抽取 |
| IFRA 法规 | IFRA 官网公开标准 | RAG 检索，配方合规自动检查 |

模块 2：多 Agent 评香智能体

这是核心亮点。不是单一 prompt，而是设计多个专业 Agent 协作：

主控 Agent → 调度
  ├─ 香调拆解 Agent：Top / Heart / Base notes 识别
  ├─ 持久度评估 Agent：预估留香时长（与你在 DSM 的微胶囊经验直接关联！）
  ├─ 扩散性评估 Agent：香氛投射力判断
  ├─ 消费者偏好预测 Agent：基于人群特征预测接受度
  └─ 安全合规 Agent：IFRA 限用物质自动检查

技术选型：
- Agent 框架：LangGraph 或 CrewAI，定义有向无环任务图
- LLM 选择：GPT-4o / Claude Sonnet（评香需要强语言能力）
- 工具调用：每个 Agent 可调用 RAG 检索、计算工具、法规查询

模块 3：RAG 检索增强

- 配方库：数千条公开香精配方 → chunk → embedding → ChromaDB / FAISS
- 法规库：IFRA 标准文档 → 支持"某某原料在 Category 4 产品中的限量是多少"类型查询
- 案例库：经典香水评测 → 作为 few-shot 示例注入 prompt

模块 4：LLM-as-Judge 评估框架（最关键）

这是大厂面试官最看重的部分——你怎么证明系统是有效的？

评测维度：
1. 香调识别准确率：LLM 判断 vs Fragrantica 用户投票的 Top/Mid/Base notes
2. 评分一致性：LLM 打分 vs 人类评香师打分（Kendall τ / Spearman ρ）
3. 文案质量：人工评估或 LLM-as-Judge（流畅性、专业性、信息量）
4. 消融实验：
   - w/o RAG → 幻觉率上升多少？
   - w/o Agent 协作 → 评测质量下降多少？
   - w/o Few-shot → 准确率下降多少？
