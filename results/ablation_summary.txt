第二部分：LLM评审评估（关键发现）
═══════════════════════════════════════════════════════════

LLM评审对RAG预测与Fragrantica真实数据在4个维度上的评分
（1-5分制，30个样本）：

维度                      预测分    真实分    差距
─────────────────────────────────────────────────────
香调准确性                  4.37      3.87    -0.50
层次正确性                  4.13      3.87    -0.27
连贯性                      4.37      3.90    -0.47
专业度                      4.37      3.80    -0.57
─────────────────────────────────────────────────────
总体                        4.37      3.82    -0.55


关键发现：LLM评审更偏好我们的预测结果，而非真实数据

  预测总体得分：4.37/5  ← 更高
  真实数据得分：3.82/5  ← 更低

  此外：F1与评审得分之间的相关系数 = -0.139（弱负相关）
  → 词元重叠度和质量评估衡量的是不同维度

这为什么重要：
  1. Fragrantica数据来自众包，并非由调香师整理。
     真实香水有时存在非常规或不一致的香韵配置，
     专业评审会对此指出来。

  2. 我们的RAG系统产出的预测比其训练所用的真实数据
     更加标准化、更接近专业水准。LLM评审一致认为
     我们的输出结构更优。

  3. 较低的F1（0.40）部分是由于真实数据存在噪声，
     而不仅是模型能力不足。模型在学习结构化的调香知识，
     而非机械记忆众包标签。

  4. 这正是LLM评审对主观领域至关重要的原因：
     精确匹配指标会惩罚合理但富有创造性的答案。

第三部分：研究发现总结
═══════════════════════════════════════════════════════════

1. RAG是影响力最大的单一技术（精确+20.5%，模糊+12.7%）
   从24k香水数据库动态检索提供了LLM内部无法生成的
   相关上下文。

2. 少样本单独使用有效（+6.8%），但与RAG结合则冗余（-0.2%）
   静态示例在独立使用时有所帮助，但RAG的动态检索以更好的
   查询相关性实现了相同的功能。

3. 多智能体分解对本任务有害（-14.8%）
   本任务是知识检索，而非多步推理。
   增加智能体只会增加错误传播，不会带来价值。

4. 最优架构：单次提示词 + RAG（精确F1=0.406，模糊F1=0.443，评审=4.37/5）
   简单、高效，被评为"接近专业水平"。

5. 仅看F1指标会误导对主观任务的判断
   LLM评审显示，预测结果在质量维度上往往优于真实数据，
   尽管词元重叠度不高。

6. 模糊匹配提升了所有方法（+9~17%），但排名保持稳定
   该指标修正验证了模型比旧F1数据所反映的更强，
   且无需修改任何模型代码。

第四部分：模糊香韵匹配 — 评估指标修正
═══════════════════════════════════════════════════════════

问题：旧版F1使用精确字符串匹配。
  "cedarwood" ≠ "cedar" — 0分，尽管它们是同一种成分。
  "italian bergamot" ≠ "bergamot" — 0分。
  这惩罚了给出实质上正确回答的模型。

解决方案：utils.py中的3层模糊匹配回退机制

  第1层 — 硬编码同义词映射（约120条）：
    对最常见变体的人工编写规则。
    "cedarwood" → "cedar"、"sandal" → "sandalwood"、
    "white musk" → "musk"、"vetivert" → "vetiver"、
    "olibanum" → "frankincense"、"cassis" → "black currant"

  第2层 — LLM构建的同义词表（1,650条）：
    DeepSeek将1,655个独特香韵聚类为1,421个规范组。
    151个组具有2个以上变体。
    例如："agarwood (oud)" → "agarwood"、"acácia" → "acacia"

  第3层 — 基于规则的规范化：
    去除地理前缀："italian bergamot" → "bergamot"
    去除颜色修饰词："white musk" → "musk"（含受保护短语列表）
    去除后缀："rose absolute" → "rose"、"vanilla extract" → "vanilla"

  最终回退 — Levenshtein编辑距离（阈值=2）：
    捕获剩余的近似拼写错误/拼写变体。
    "graperfruit" → "grapefruit"（编辑距离=1）

关键设计决策：硬编码映射优先于规则检查。
  防止"white musk"被规则误剥离为"musk"（当已有显式映射时）。
  同时防止"black currant"被错误剥离为"currant"（受保护短语列表）。

结果（所有方法已重新评估，2026-05-07）：

方法                      精确F1 → 模糊F1     变化量
────────────────────────────────────────────────
频率法                     0.299   →   0.327    +9.4%
LLM零样本                  0.337   →   0.393    +16.6%   ← 提升最大
LLM + RAG                  0.406   →   0.443    +9.1%    ← 绝对值最佳
LLM + RAG + 少样本         0.405   →   0.442    +9.1%    ← 冗余

RAG各层（模糊匹配）：
层次      精确 → 模糊    增量
────────────────────────────────
前调      0.300 → 0.311    +0.011  (3.7%)
中调      0.287 → 0.317    +0.030  (10.5%)
基调      0.431 → 0.475    +0.044  (10.2%)
总体      0.406 → 0.443    +0.037  (9.1%)

关键观察：
  1. LLM零样本从模糊匹配中获益最大（+16.6%）
     → 未经辅助的LLM使用更多创造性/多样化的香韵名称
     → 模糊匹配正确地规范化了这些变体
  2. RAG获益较少（+9.1%），因为参考资料标准化了输出
     → LLM从检索到的香水中学习规范名称
     → 但RAG在F1绝对值上仍然胜出（0.443 vs 0.393）
  3. 基调在所有方法中获益最为一致
     → 木质、麝香、树脂类具有最多的修饰词变体
  4. 少样本的冗余性在两种指标下均得到确认
     → 0.406 vs 0.405（精确），0.443 vs 0.442（模糊）
  5. RAG仍然是最佳单一技术，目前F1=0.443

这证明了什么：
  - 精确匹配F1在领域特定评估中存在根本缺陷
  - 词元重叠度 ≠ 语义正确性（与LLM评审的教训相同）
  - RAG模型比旧版F1数值所反映的约有9%的提升
  - 模糊匹配提升了所有方法，但相对排名保持稳定

 1. LLM + RAG（DeepSeek API）          F1 = 0.443   ★ 绝对值最佳
 2. LLM + RAG + 少样本                 F1 = 0.442   相对于RAG冗余
 3. 多智能体编排                       F1 = 0.418   复杂但更差

关键发现：
  1. 微调模型优于LLM零样本（模糊+6.4%）
     但无法匹敌RAG（模糊-5.6%），因为RAG使用24k香水数据库的
     实时检索作为参考。

  2. 微调模型与多智能体并列（0.418 vs 0.418）
     但使用1次推理调用，而非5次调用 + LangGraph开销。

  3. 基调是最佳层次（模糊F1=0.415）— 木质、麝香、
     树脂在训练数据中具有更清晰的香调→香韵映射关系。

  4. 前调最弱（F1=0.257）— 前调通常为柑橘/清新调，
     高度可变且受香调约束较少。

  5. 微调模型每层预测的香韵数量较少（平均3个），
     而LLM始终预测5个，这限制了召回率。训练数据中的
     香韵数量自然变化，因此模型学习了真实分布。

  6. 本地且免费 — 无需API密钥、无延迟、无使用限制。
     非常适合演示、快速迭代和离线使用。

  7. 0.418的F1与多智能体具有竞争力，但实现复杂度
     远低于后者（单次前向传播 vs 编排）。
